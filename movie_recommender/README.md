# Endee Vector DB: Semantic Movie Recommender

This project was built to demonstrate a practical and advanced use case of the [Endee Vector Database](https://github.com/endee-io/endee) for the internship evaluation process.

## 🎯 Project Overview & Problem Statement
Traditional movie recommendation systems often rely on basic metadata overlapping (e.g., matching "Action" genres) or collaborative filtering. However, these systems fall short when a user wants a movie with a specific *thematic* or *plot-driven* vibe (e.g., "A sci-fi plot where they travel back in time to save humanity").

**Solution:** This application provides a **Semantic Search Recommendation System**. It embeds the natural language plot overviews of thousands of movies into high-dimensional vectors. When a user describes what they want to watch, their query is also embedded, and the Endee Vector Database instantly performs a cosine similarity search to find the closest matching movies based purely on plot semantics.

## 🏗 System Design & Technical Approach

The overall architecture involves:
1. **Data Source:** We utilize a cleaned Movie Database (TMDb) dataset from HuggingFace containing movie titles, overviews, ratings, and genres.
2. **Embedding Model:** `sentence-transformers/all-MiniLM-L6-v2` runs locally to generate dense 384-dimensional vector embeddings for both the movie plots (during ingestion) and user queries (during search).
3. **Vector Database (Endee):** The open-source Endee database engine serves as the core vector storage and retrieval layer, utilizing an `INT8` precision cosine-distance index for high performance.
4. **Backend (FastAPI):** A fast, asynchronous Python API handles incoming search requests from the frontend, vectorizes them, and queries the Endee DB.
5. **Frontend (HTML/JS/CSS):** A visually stunning, responsive user interface built using vanilla web technologies, showcasing a modern dark aesthetic with glassmorphism to interact with the API effortlessly.

## 🧠 How Endee is Used

Endee sits at the very core of this application's performance.
- **Index Creation:** We create an index named `movies` with a dimension of `384` and `cosine` distance metric. We also implement `Precision.INT8` to demonstrate Endee's ability to optimize memory while maintaining retrieval accuracy.
- **Vector Upsertion:** During the ingestion phase (`indexer.py`), we serialize the vectors generated by the HuggingFace model along with extensive movie metadata (Title, Overview, Genres, Year) and upsert them into Endee in large batches.
- **Low-Latency Querying:** In the FastAPI route (`main.py`), Endee's `query` method is invoked to instantly return the `top_k=6` closest semantic matches along with the stored metadata, which is fed directly to the UI.

## 🚀 Setup and Execution Instructions

Follow these exact steps to run the complete project on your local machine.

### Prerequisites
- Docker & Docker Compose (for running Endee easily)
- Python 3.9+
- Git

### Step 1: Start the Endee Vector Database
Navigate to the `movie_recommender` directory and start Endee using Docker Compose:
```bash
cd movie_recommender
docker-compose up -d
```
*Endee will now be running and listening on port 8080.*

### Step 2: Set up the Python Environment
Create a virtual environment and install the required dependencies:
```bash
python -m venv venv

# On Windows:
.\venv\Scripts\activate
# On macOS/Linux:
source venv/bin/activate

pip install -r requirements.txt
```

### Step 3: Ingest Data into Endee
To populate the database, run the ingestion script. This will download a sample dataset from HuggingFace, embed the movie plots, and insert them into Endee.
*(Note: Downloading the model and dataset may take a minute or two depending on your internet connection).*
```bash
python indexer.py
```
You should see logging indicating batches being upserted into Endee.

### Step 4: Start the Web Application
Once ingestion is complete, start the FastAPI backend server:
```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

### Step 5: Test the Application
Open your web browser and navigate to:
**[http://localhost:8000](http://localhost:8000)**

Try queries like:
- *"A futuristic movie where robots take over the world"*
- *"A funny romantic comedy set in New York"*
- *"An intense psychological thriller about a detective finding a serial killer"*
